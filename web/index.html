Signal generation system

Batch ingestion → PostgreSQL → request-time scoring

Signals are quantitative model outputs, not investment advice. Outcomes are not guaranteed. Short selling requires broker borrow/locate; the API only labels position_intent.

Architecture

Polygon supplies prices and fundamentals; Perigon supplies news. Jobs run on a schedule or manually, write to PostgreSQL (Timescale for OHLCV), and optionally publish to Kafka. The signal API is a long-running HTTP server: each request recomputes scores from the latest database state.

Ingestion order

Suggested first-time batch sequence (from services documentation):

  1. 01
    universe_cron
    Builds symbols, universe snapshots, filtered universe; liquidity and market-cap filters; sector and benchmark ETF fields from Polygon.
  2. 02
    price_ingest
    Daily aggregates for universe symbols and benchmark ETFs (SPY, QQQ, sector ETFs); extends lookback for beta and attribution.
  3. 03
    technical_engine
    RSI, MACD, Bollinger, VWAP on daily bars; regime snapshot from SPY and VIX.
  4. 04
    news_ingest
    Perigon articles, dedupe, match tickers; requires symbols from the universe.
  5. 05
    nlp_worker
    FinBERT sentiment for articles missing scores (ML extra).
  6. 06
    fundamentals_ingest
    Polygon financials into fundamentals_snapshot and heuristic fundamental_score.
  7. 07
    attribution_job
    5d move vs SPY, rolling beta, sector ETF, peer percentile → attribution_snapshot (explanatory).
  8. 08
    sector_sentiment_job
    Sector-level weighted news vs benchmark ETF returns → sector_sentiment_snapshot (after NLP and prices).
  9. 09
    signal_api
    GET /v1/signals (keyed) or GET /public/v1/signals when enabled — reads latest rows, no per-request training.

Optional: impact_job and source_scoring refine publisher weights for sentiment aggregation.

Request-time computation

Each call loads current technicals, aggregates 14d news sentiment (publisher-weighted), blends with optional fundamentals, applies regime dampening to positive conviction, then classifies action and position intent.

technical_z_score ──┐ news sentiment agg ─┼── blend_scores → apply_regime → classify_action_intent fundamentals (opt) ─┘
